Bricklink

Introduction

Bricklink is a lego marketplace. Essentially it is the EBay for lego, where you can buy or sell anything to do with Lego. The lego community maintain a number of fantastic resources that catalogue all of the Lego products, often down to the detail of the number, colour and type of each part in a lego set.

The bricklink website is backed by one such catalogue and it available for free download in both TSV and XML formats. This dataset consists of an RDFization of that data to make it available as Linked Data and accessible via the Kasabi APIs. The data has been enhanced to include EANs (product identifiers) for many of the sets.

Currently the dataset consists of descriptions of over 9000 Lego Sets and over 23,000 Lego parts.

Potential Uses

  • Building a lego set browser
  • Building a lego inventory manager
  • Building a lego project or set recommender based around identifying sets that someone may be able to create with existing parts
  • As the core of a larger dataset that aggregates additional information, e.g. inventories of custom lego models produced by the community.

Vocabularies

The full set of vocabularies used is as follows:

NamePrefixURINotes

Dublin Core Termsdcthttp://purl.org/dc/terms/

The sections below outline which properties from these vocabularies have been used on the various entities that can be found in the dataset.

For readability, in the tables below, property URIs are given using their prefixed names, e.g. brick:weight refers to: http://data.kasabi.com/dataset/bricklink/schema/weight.

Book (brick:Book)

Lego have produced a range of lego related books. Each of these books in the dataset are modelled with a type of brick:Book.

The URIs for the Book have been constructed using the following URI pattern:

http://data.kasabi.com/dataset/bricklink/book/{id}

Where id is the unique code for the book in the lego part database

The following properties are associated with Book:

Property NameProperty URINotes
Categorybrick:categoryURI of the lego category associated with the book.
Dimension Xbrick:dimensionXWidth of the book
Dimension Ybrick:dimensionYHeight of the book
Dimension Zbrick:dimensionZDepth of the book
Identifierdct:identifierLego part database identifier for the book
Labelrdfs:labelTitle of the book
Publisheddct:publishedYear that the book was published
Weightbrick:weightWeight of the book

The following example demonstrates how these properties are used to describe a Book:

  @prefix brick: <http://data.kasabi.com/dataset/bricklink/schema/> .
@prefix dct: <http://purl.org/dc/terms/> .
@prefix foaf: <http://xmlns.com/foaf/0.1/> .
@prefix owl: <http://www.w3.org/2002/07/owl#> .
@prefix rdfs: <http://www.w3.org/2000/01/rdf-schema#> .
@prefix skos: <http://www.w3.org/2004/02/skos/core#> .

<http://data.kasabi.com/dataset/bricklink/book/1030b16> 
   brick:category <http://data.kasabi.com/dataset/bricklink/category/576>;
   brick:dimensionX "21";
   brick:dimensionY "15";
   brick:dimensionZ "0.02";
   brick:weight "15";
   dct:identifier "1030b16";
   dct:published "1983";
   a brick:Book;
   rdfs:label "Set 1030 Activity Booklet 16 - Lifting with Pulleys" .

Category (brick:Category)

Lego parts, sets, themes and books, etc are all organized into a number of broad categories or themes. Each of the Category in the dataset are modelled with a custom type of brick:Category, but are also typed with the more commonly used skos:Concept class. The lego categories/themes thereby consist of a SKOS Concept Scheme which has been given the URI of:

http://data.kasabi.com/dataset/bricklink/category

The URIs for the Category have been constructed using the following URI pattern:

http://data.kasabi.com/dataset/bricklink/category/{id}

The following properties are associated with Category:

Property NameProperty URINotes
Broaderskos:broaderURI of a parent category
Identifierdct:identifierUnique identifier for the category
In Schemeskos:inSchemeReference to the concept scheme
Narrowerskos:narrowerURI of a child category
Pref Labelskos:prefLabelPreferred label of the categoy

The following example demonstrates how these properties are used to describe a Category:

  @prefix brick: <http://data.kasabi.com/dataset/bricklink/schema/> .
@prefix dct: <http://purl.org/dc/terms/> .
@prefix foaf: <http://xmlns.com/foaf/0.1/> .
@prefix owl: <http://www.w3.org/2002/07/owl#> .
@prefix rdfs: <http://www.w3.org/2000/01/rdf-schema#> .
@prefix skos: <http://www.w3.org/2004/02/skos/core#> .

<http://data.kasabi.com/dataset/bricklink/category/576> 
   dct:identifier "576";
   a brick:Category,
   skos:Concept;
   skos:inScheme <http://data.kasabi.com/dataset/bricklink/category>;
   skos:prefLabel "Curriculum Activity Book" .

Inventory (brick:Inventory)

Some lego sets have inventories associated with them. Each of the inventories in the dataset are modelled with a type of brick:Inventory. An inventory is broken down into a list of Inventory Items (brick:InventoryItem).

The URIs for the Inventory have been constructed using the following URI pattern:

http://data.kasabi.com/dataset/bricklink/set/{id}/inventory

Where id is the identifier for the lego set.

The following properties are associated with Inventory:

Property NameProperty URINotes
Has Partdct:hasPartReference to a inventory item brick:InventoryItem that makes up the inventory
Setbrick:setURI of the set that this inventory is associated with.

The following example demonstrates how these properties are used to describe a Inventory:

  @prefix brick: <http://data.kasabi.com/dataset/bricklink/schema/> .
@prefix dct: <http://purl.org/dc/terms/> .
@prefix foaf: <http://xmlns.com/foaf/0.1/> .
@prefix owl: <http://www.w3.org/2002/07/owl#> .
@prefix rdfs: <http://www.w3.org/2000/01/rdf-schema#> .
@prefix skos: <http://www.w3.org/2004/02/skos/core#> .

<http://data.kasabi.com/dataset/bricklink/set/00-3/inventory> 
   brick:set <http://data.kasabi.com/dataset/bricklink/set/00-3>;

Inventory Item (brick:InventoryItem)

Inventory items describe the relationship between a part, its colour and number in a particular lego set inventory. Each of the Inventory item in the dataset are modelled with a type of brick:InventoryItem.

The URIs for the Inventory item have been constructed using the following URI pattern:

http://data.kasabi.com/dataset/bricklink/set/{set}/inventory/{part}

The following properties are associated with Inventory item:

Property NameProperty URINotes
Colourbrick:colourURI of a lego brick colour
Is Part Ofdct:isPartOfInventory of which the item is a part
Is Primary Topic Offoaf:isPrimaryTopicOf
Itembrick:itemURI of the lego item that is included in the inventory. Could be a part, book, or even set.
Labelrdfs:labelLabel for the inventory item
Valuehttp://www.w3.org/1999/02/22-rdf-syntax-ns#value Number of parts

The following example demonstrates how these properties are used to describe a Inventory Item:

  @prefix brick: <http://data.kasabi.com/dataset/bricklink/schema/> .
@prefix dct: <http://purl.org/dc/terms/> .
@prefix foaf: <http://xmlns.com/foaf/0.1/> .
@prefix owl: <http://www.w3.org/2002/07/owl#> .
@prefix rdfs: <http://www.w3.org/2000/01/rdf-schema#> .
@prefix skos: <http://www.w3.org/2004/02/skos/core#> .

<http://data.kasabi.com/dataset/bricklink/set/00-1/inventory/29c01-1> 
   brick:colour <http://data.kasabi.com/dataset/bricklink/color/1>;
   brick:item <http://data.kasabi.com/dataset/bricklink/part/29c01>;
   dct:isPartOf <http://data.kasabi.com/dataset/bricklink/set/00-1/inventory>;
   a brick:InventoryItem;
   <http://www.w3.org/1999/02/22-rdf-syntax-ns#value> "6"^^<http://www.w3.org/2001/XMLSchema#integer>;

Data Conversion and Source Code

The data is sourced directly from the Bricklink website via their download forms.

The code to support the conversion of this dataset has been open sourced. Developers interested in collaborating to improve the modelling or fix up data conversion errors, can contribute to the project via github.

http://github.com/ldodds/bricklink
